🔑 GitHub Personal Access Token 创建指南

⚠️ 重要提醒

GitHub 已不再支持密码登录，必须使用 Personal Access Token 才能推送代码。Token 只显示一次，务必立即复制保存！

📋 操作步骤

步骤 1：访问 Token 创建页面

点击下面的按钮打开 GitHub Token 创建页面：

🚀 打开 Token 创建页面

如果未登录，会跳转到登录页面，输入：

  • Username: seray1980
  • Password: [你的 GitHub 密码]
步骤 2：进入 Tokens 页面

如果需要手动导航：

  • 点击右上角头像
  • 选择 Settings（设置）
  • 在左侧菜单中向下滚动
  • 找到 Developer settings
  • 点击它
  • 在左侧选择 Personal access tokens
  • 点击 Tokens (classic)
步骤 3：生成新 Token

在 Tokens 页面，点击右侧的按钮：

Generate new token (classic)

步骤 4：配置 Token

1. Note（备注）必填

输入：autotrade-mobile

2. Expiration（过期时间）必填

选择：No expiration（永不过期）

或根据需要选择 30/90 天

3. Select scopes（选择权限）必填

找到 repo 选项，勾选它

⚠️ 勾选 repo 会自动勾选所有子选项，这是必需的权限

其他选项不要勾选！

4. 点击生成

滚动到页面底部，点击：Generate token

步骤 5：复制 Token最重要！
⚠️ Token 只显示一次！

生成成功后，页面顶部会显示 Token，必须立即复制！刷新页面后就看不到了！

Token 格式示例：

ghp_1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6p7q8r9s0t

共 40 个字符，以 ghp_ 开头

如何复制：

  • 方法 1：点击 Token 右侧的复制图标 📋
  • 方法 2：手动选择整个 Token，按 Ctrl+C

保存 Token：

立即将 Token 保存到安全的地方：

  • 打开记事本
  • 粘贴 Token
  • 保存到桌面：github_token.txt
步骤 6：使用 Token

当你在 Git 中推送代码时：

git push -u origin main

会提示：

Username for 'https://github.com': seray1980
Password for 'https://seray1980@github.com':

输入：

  • Username: seray1980
  • Password: 粘贴你的 Token（不是 GitHub 密码！）
⚠️ 重要

Token 不是 GitHub 密码！推送时密码框输入的是 Token！

✅ 完成！

现在你有了一个可用的 Personal Access Token，可以继续推送代码了！

运行 git_deploy.bat 来推送代码，推送时输入你的 Token。

❓ 常见问题

Q: Token 复制失败怎么办？

A: 尝试使用复制按钮，或手动选择整个 Token 后复制。确保 Token 完整（40个字符）。

Q: Token 不工作？

A: 检查 Token 是否完整，确认勾选了 repo 权限，检查 Token 是否过期。如果还不行，重新创建 Token。

Q: 忘记保存 Token 怎么办？

A: Token 只显示一次，需要重新创建。访问：https://github.com/settings/tokens

Q: Token 可以分享吗？

A: ❌ 不可以！Token 相当于密码，分享会导致账户被盗。

🔐 Token 安全
  • ✅ 保存在安全的地方
  • ✅ 可以给不同项目创建不同 Token
  • ✅ 定期更换 Token
  • ✅ 如果 Token 泄露，立即删除
  • ❌ 不要分享给他人
  • ❌ 不要上传到代码仓库
  • ❌ 不要写在代码中
